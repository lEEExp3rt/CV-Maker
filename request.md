# CV-Maker 需求清单

现在我有一个需求，我想基于前端框架制作一份互联网大厂技术岗的校招或者实习求职的简历。

## Brief

### Contents

简历需要大致包含以下内容（“可选”意为可配置项，由用户决定是否启用）：

- 个人基本信息
  - 姓名
  - 邮箱
  - 电话号码
  - 出生年月（可选，默认无）
  - 个人主页链接（可选，默认无）
  - 其他社交媒体如LinkedIn、GitHub（可选，默认无，需要支持自定义补充）
  - 个人照片（可选，默认无）
- 教育背景
  - 学校中英文名称
  - 学位
  - 专业名称
  - 就读时间
  - gpa和排名（可选，默认无）
  - 核心课程（可选，默认无）
- 实习经历
  - 实习单位中英文名称（英文可选，默认无）
  - 实习起止时间
  - 实习岗位名称
  - 实习工作分条目要点
- 项目背景
  - 项目名称（英文可选，默认无）
  - 项目角色
  - 项目起止时间
  - 项目内容分条目要点
- 专业技能（可选，默认无）
- 获奖与荣誉（可选，默认无）

### Styles & Formats

排版和样式可以参考常见的校招求职简历的样式，不需要很高级或者花哨但是需要保证简洁和美观。同时，需要注意的是排版完转化成pdf后最好能够在一页内容纳简历（除非使用者自身添加了非常大量的简历内容导致溢出多页），这是校招和简历的基本要求。基于这个要求，我们需要保证简历样式和排版紧凑，但是不能为了实现这个而把字体修改成非常小或者行间距非常小导致页面看上去过于紧凑狭小而导致不美观。所以我们要主打一个简洁美观，减少冗余信息。

标题应为个人姓名（层级最高），各模块名称为小标题（层次第2），各段经历如教育经历和实习经历等的标题为层级第3，一次类推，不同层级的标题应该有不同的字号大小

字体选择上，请使用比较美观舒适的中英文字体，并能够支持自定义配置字体

颜色方案上，请提供基础的1-2种配色方案，并能够支持二次开发和自定义

### Output

简历最终版本应该是PDF格式，所以本项目的最终输出需要是一个PDF文件。

## Advanced Requirements

### Architecture

首先，这应该是一个完整的代码库，一个完整的简历制作项目。这个项目应该设计并实现内容、样式两大板块分离的架构，样式部分是负责组织简历外观、排版和样式用的，内容部分由使用者完成填充。

### Tech Stack

请基于 React 框架实现，并使用 Vite 进行构建

### Development & Build

请你为这个项目提供 Docker 容器，容器请存放在 `.devcontainer/` 目录下，容器文件为 Dockerfile，入口文件为 entrypoint.sh，VSCode 开发容器文件为 `devcontainer.json`

### Better Usage

为了做到开箱即用，给使用者最佳的使用和上手体验，最好能够对使用者屏蔽技术细节，做到即使使用者不懂前端但仍可以通过简单配置和简单命令完成制作。我希望能够把所有的用户填充内容实用yaml文件或者JSON文件完成编写，这样用户只需要填写他们自己的内容到几个yaml或者JSON文件中然后一键执行生成命令就可以完成简历制作，而无需精通前端。

具体来说，项目启动后，应提供一个开发服务器 (npm run dev)，用户可实时预览简历效果。当用户修改yaml文件后，页面应自动热更新。最终，用户通过 npm run build 或 npm run export 命令，生成最终的PDF文件到 dist/ 目录

```yaml
# 设计样例
cv:
    personal_info:
        name: 张三
        phone: +86-xxxx-xxxx-xxxx
        email: example@mail.com
        homepage:
        image: contents/images/me.png
    education:
        - school: "XXX University"
          major: "Computer Science"
    ...
```

如有必要，也可以拆成多个内容模块，然后由一个根内容文件控制

### Documents

完成代码库的编写后，请编写完整的文档，包括

- 仓库自述文档，介绍本项目主要内容、环境搭建和基础使用方法等
- 配置说明文档，介绍本项目所有的配置信息，告诉开发者和使用者如何配置项目
- 开发文档，提供二次开发的相关必要信息，告诉开发者如何修改某某功能
- 其他必要的文档
